# 目标

构建一个支持登录认证与移动端适配的任务看板 Web 应用（demo-web），帮助用户高效管理和跟踪任务进度。

# 需求

- 用户可通过账号密码完成登录，登录态持久化（如 Token 存储），支持退出登录
- 提供任务看板页面，支持按状态分列展示任务（如：待处理 / 进行中 / 已完成）
- 用户可在看板上创建新任务，填写任务标题、描述、优先级等基本信息
- 用户可拖拽或操作按钮将任务在不同状态列之间移动
- 用户可编辑或删除已有任务
- 页面完整适配移动端（响应式布局），在主流手机浏览器下可正常使用
- 项目以 `npm run build` 构建，产物通过 Docker 镜像（demo-web）以 80:80 端口对外服务
- 代码托管于 GitHub `yundongfusheng/demo-web`，主分支为 `main`

# 验收标准

- **登录流程**
  - Given 用户访问应用根路径，When 未登录，Then 自动跳转至登录页
  - Given 用户输入正确账号密码，When 点击登录，Then 跳转至任务看板页并保持登录态
  - Given 用户输入错误账号密码，When 点击登录，Then 页面显示明确错误提示，不跳转

- **任务看板**
  - Given 已登录用户进入看板，When 页面加载完成，Then 展示至少三个状态列，并正确渲染已有任务卡片
  - Given 用户点击「新建任务」，When 填写必填信息并提交，Then 新任务卡片出现在对应列中
  - Given 用户对某任务执行状态变更操作，When 操作完成，Then 任务卡片移动至目标列，数据持久化
  - Given 用户对某任务执行删除，When 确认删除，Then 任务从看板中消失

- **移动端适配**
  - Given 用户使用宽度 ≤ 375px 的设备访问，When 页面渲染，Then 布局无横向溢出，按钮与文本可正常点击和阅读
  - Given 用户在移动端操作任务，When 执行创建/状态变更，Then 功能与桌面端一致可用

- **构建与部署**
  - Given 在项目根目录执行 `npm run build`，When 构建完成，Then 无报错，产物输出至 dist 目录
  - Given Docker 镜像 `demo-web` 启动并映射 80:80，When 访问 `http://localhost`，Then 应用正常加载并可使用全部功能
